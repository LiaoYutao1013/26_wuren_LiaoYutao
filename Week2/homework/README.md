# Week2 学习记录

### 作业1：遗传算法

搜索启发式算法时，得知自己之前学的贪心算法也是一种启发式，有点意外，但细想一下发现它确实符合这个标准。设计启发式算法更多是为了在条件受限的情况下解决实际问题，因而相当考验设计者的水平，需要找到一套规则来让输出趋向于需要的答案。
遗传算法、模拟退火算法...还有很多启发式算法，大多取材于现实世界的事件，设计这块我想还要向实际多多取经才是。
这次着重学了遗传算法，用二进制编码去搜索二维函数的最优解，把“选择、交叉、变异”这些生物学中的概念同算法、代码联系起来，用基因序列等效解决实际问题需要采用的解。阅读代码的过程中，我意识到遗传算法能不能正常工作的关键在于编码方式、适应度函数和解码过程的构建是否合适。就示例而言，香蕉函数的参数（我指的是那个函数里的常数）为何这样选取还在琢磨，也许是被反复验证过优化速度快而直接拿来用了？
`genetic_algorithm.py` 里原先函数体内注释较少，我补充了不少，具体在代码里查阅。
出于设备还未排查清楚的故障，我先把主函数的输出改成了图片，方便我可视化预览结果。

### 作业2：多元线性回归与岭回归

最有收获的地方是理解了正则化的作用：它不是为了追求更复杂的模型，而是为了抑制系数过大、提升泛化能力。通过对比系数变化和 RMSE，我对“偏差-方差权衡”有了更直观的认识。

### 作业3：SVM 分类

依旧是经典的 Iris 数据集。因为之前已经做过回归任务，这次更关注分类器的决策边界、特征尺度和调参过程。实践中最明显的一点是，SVM 对特征尺度很敏感，所以标准化几乎是必备步骤(这一点和之前数模评估生物量走向时做归一化倒是一样)。之后再用 `GridSearchCV` 对 `C` 和 `gamma` 做简单搜索，可以直接看到参数变化对结果的影响。

大致总结了一下分类的流程：数据集观察$\rightarrow$降维可视化$\rightarrow$标准化$\rightarrow$训练$\rightarrow$评估$\rightarrow$再调参。

### 作业4：K-means 聚类

K-means 和前面的监督学习不一样，它没有标签可直接学习，所以更考验对算法迭代逻辑的理解。这次任务我主要补完了初始化中心、样本分配、中心更新、代价函数计算和收敛判断这几个模块。

最大的难点在于怎么实现对批量数据/数据文件的处理，算法本身的思想并不难理解：指定聚类重心个数（先出散点图目测？不靠目测也能确定重心的个数吗？）   $\rightarrow$计算平均距离（这里涉及对“距离”的定义理解）$\rightarrow$调整聚类重心位置（理想的话应该沿梯度方向调位置，尽量减少迭代次数）$\rightarrow$重复如此，直到输出达到预期。

## 这周的收获

1. 算法理解不应该停留在会用，而要和数据处理相结合。
2. 标准化、训练集/测试集划分、验证集选择这些步骤，往往比调用模型本身更重要。
3. 可视化可以便于理解算法，并直观感受到算法的执行结果（以及优化调参的过程），尤其是在遗传算法、回归曲线、SVM 分类和 K-means 聚类里。

# 深度学习笔记（第三次考核 - 基础任务二）

对课内概念的研究：

## 1.正则化

最终目的：尽可能避免模型出现过拟合。
(过拟合的概念在下文中有探索)

正则化的大方向：适当降低模型复杂度，让模型去学习数据中更本质、更通用的特征，而不是过分迎合训练集的数据。
以下是搜集到的一些常见正则化方法：

### 1. L1 与 L2 正则化（权重衰减）

通过在损失函数（Loss Function）后面加一个“惩罚项”，让模型不会把权重练得过大。

1. **L2 正则化 (Ridge Regression)：**
公式：$J_{new} = J_{old} + \frac{\lambda}{2n} \sum w^2$
效果： 倾向于让权重均匀地变小，使模型更平滑。它是深度学习中最常用的正则化手段。

1. **L1 正则化 (Lasso Regression)：**
公式：$J_{new} = J_{old} + \frac{\lambda}{n} \sum |w|$
效果：倾向于让不重要的权重直接变成0。这会产生稀疏性，相当于自动帮你做特征选择。

### 2. Dropout（随机失活）

在训练过程中，随机让一部分神经元“罢工”（输出设为 0），这样在一定程度上可以防止每个神经元学习到的特征过于片面。

### 3. 数据增强 (Data Augmentation)

过拟合是因为数据不足或模型过于复杂，解决这个问题的思路除了对模型动手，从输入数据端处理，增强数据同样也是一种解决思路。

方法：对图像进行旋转、翻转、缩放、调曝光、对比度、饱和度······在不破坏图像本质特征的基础上变化图像，让模型泛化能力变强。
思想：只要模型见过足够多“奇形怪状”的猫，它就不会因为照片里猫的角度不对而认不出来。
但增强要有限度。早年读过一些报道，在正常图片中加入过量的噪声（噪点）过后，模型可能会识别出完全不符合预期的结果（如企鹅被识别成狗），这也对模型本身的鲁棒性有一定要求。

### D. 早停法 (Early Stopping)

方法：监控模型在验证集上的表现。当发现验证集的误差不再下降反而上升时，果断停止训练。

**注意**：正则化过度，可能会将模型推向欠拟合（所谓物极必反大抵如此）

## 2.过拟合/最佳拟合/欠拟合

模型训练的过程大抵是在寻找一组最佳参数（或者从一开始要选好模型架构）。让它既能很好地符合训练集，又能在测试集、预测集（还有实际运用）中表现优良。
而过拟合、最佳拟合、欠拟合是寻找这些参数（架构）时的状态。

### 欠拟合

原因：模型太简单，学到的特征不足以反映真实情况；
表现：训练和测试时的表现都不佳；
个人认为跟模型选取的关系更大一些，最好能在观察数据特征（可视化）后再决定怎么选用，而不是盲目决定（一眼三角函数的就避免选线性之类）

### 最佳拟合

这个时候模型充分提炼了数据的特征，理解到位；
表现：训练时表现很好；测试集、预测集成绩同样很好，泛化能力强。

### 过拟合

原因：模型太复杂，提炼了过多的细节乃至杂音，将无关信息一并纳入了模型的考量标准。
表现：训练集准确率100%（表现超级好）；但泛化能力很差。
过犹不及的形象表现，好比一组1000个点组成的数据集，能用线性拟合非要插值做1000次方的非线性模型，那么这个非线性模型毫无意义。

### 解决欠拟合的思路

1. 增加模型复杂度，比如在神经网络里多加几层，或者增加神经元的数量；
2. 挖掘更多特征，给模型提供更多有用的输入数据；
3. 减少正则化，如果正则化系数太大，模型的学习能力会被过分压抑，这时要适当调低正则化强度；

### 解决过拟合的思路

1. 做正则化，强迫模型不要关注细枝末节（做个折中，太偏向哪一侧都会导致模型性能下降）；
2. 增加数据量，让模型认识到哪些是杂音，哪些才是真正的特征；
3. 早停法：在训练时实时检查模型性能，达到稳定（至少在基础任务一里是这样的）后及早停止训练；
4. 简化模型，砍掉一些不必要的层或参数（这里又和欠拟合开始对抗了，模型构建是个关键）。

其他网络结构:

## ResNet （CNN的改进变体）

## 1. 背景

传统的CNN中有以下几个问题：

1. 梯度消失：深层网络梯度传播困难，存在梯度变化极其缓慢的情况；
2. 特征退化：网络层数过多反而会降低性能，预测准确度下降；

数学原理：
$$L = f_n(f_{n-1}(...f_1(x)))$$

反向传播时，
$$\frac{\partial L}{\partial x} = \frac{\partial L}{\partial f_n} \cdot \frac{\partial f_n}{\partial f_{n-1}} \cdot ... \cdot \frac{\partial f_1}{\partial x}$$

如果 $\frac{\partial f_i}{\partial f_{i-1}} < 1$，梯度会越乘越小，也就是层数越深，梯度衰减越严重，对模型性能影响很大。

## 2. 残差学习（ResNet的思想）

基本思想是学习残差比直接学习映射更容易。

传统网络中，
$$y = H(x)$$

而在ResNet中，
$$y = H(x) + x$$

其中 $H(x)$ 学习残差，即 $H(x) = y - x$。

数学原理：

设 $y = x + H(x)$，反向传播时，

$$\frac{\partial L}{\partial x} = \frac{\partial L}{\partial y} \left(1 + \frac{\partial H}{\partial x}\right)$$

其中每层梯度都大于1，即使 $\frac{\partial H}{\partial x}$ 很小，+1后也能保证梯度不会衰减（消失）。

## 3. ResNet 架构

### 基础残差块（Basic Block）

(此处让AI作了一个字符画)

```text
┌─ x ─┐
│     │
▼     │
Conv  │ (out_channel = in_channel)
ReLU  │
Conv  │
│     │
└─+─┬─┘ (相加)
  │ │
  ▼ ▼
  H(x) + x
    │
    ▼
  ReLU
```

数学表达式为
$$y = \text{ReLU}(x + F(x))$$
这里ReLU不只处理x，还处理了残差函数$F(x)$，达到学习残差的效果。

### 瓶颈残差块（Bottleneck Block）

用于更深的网络（50 层+），减少计算量：

```text
┌─ x ──────┐
│          │
▼          │
Conv 1×1   │ (降维，减少通道)
ReLU       │
Conv 3×3   │
ReLU       │
Conv 1×1   │ (升维，恢复通道)
│          │
└─+─┬──────┘ (相加)
  │ │
  ▼ ▼
  y + x
    │
    ▼
  ReLU
```

## 4. 改进变体

改进：将 Batch Norm 和 ReLU 放在卷积前：

```text
原始顺序：Conv → BN → ReLU → Conv → BN
改进顺序：BN → ReLU → Conv → BN → ReLU → Conv
```

## 两类混合

### Token Mixing

沿空间 token 维度混合信息，相当于让不同 patch 之间通信。

### Channel Mixing

沿通道维度混合信息，相当于在每个位置内部重组特征。

## 与 CNN / Transformer 的对比

CNN：卷积核局部滑动；高效、局部归纳偏置强；长距离关系需要堆层
Transformer：注意力全局交互；全局建模强；计算量高、需要大数据
MLP-Mixer：MLP 混合 token；结构简单；缺少局部归纳偏置

## 混合架构

现代视觉架构往往不是纯 CNN 或纯 Transformer，而是混合设计：

- ConvNeXt：用现代训练策略重新设计 CNN。
- MetaFormer：强调 token mixer 的统一框架。
- MobileViT：把轻量卷积和 Transformer 结合到移动端。
- YOLO 系列：CNN 主干 + PAN/FPN + 注意力或轻量模块。

## 数据增强

数据增强通过人工生成新的训练样本，增加数据的多样性，改进模型的泛化能力。
通常是对已有的数据集进行变形改造，扩大数据集。

## 基础增强方法

### 1. 几何变换

#### 旋转（Rotation）

参数：

- 旋转角度：±5° 到 ±30°（推荐 ±15°）
- 对旋转不敏感的场景（如竖拍）可增大范围

#### 缩放（Scaling）

参数：

- 缩放范围：0.8 到 1.2（推荐）
- 帮助模型学习多尺度目标

#### 裁剪（Cropping）

参数：

- 裁剪比例：0.7 到 1.0
- 可能丢失目标，谨慎使用

#### 翻转（Flipping）

参数：

- 水平翻转：通常总是使用（概率 0.5）
- 垂直翻转：取决于场景（如行人检测不推荐）

#### 透视变换（Perspective）

应用：自动驾驶（不同视角的车辆）

### 2. 颜色增强

#### 亮度调整

参数：

- 范围：0.7 到 1.3
- 模拟不同光照条件

#### 对比度调整

参数范围：0.8 到 1.2

#### 色调调整（HSV 空间）

参数：

- H 范围：±0.5
- S 范围：×0.7 到 ×1.3
- V 范围：×0.7 到 ×1.3

#### 高斯模糊

应用：模拟运动模糊、焦外

## 高级增强方法

### Cutout

移除图像的随机部分，增强模型的鲁棒性。

参数：

- 洞的数量：1-3
- 洞的大小：32×32 到 64×64

优势：

- 增强对遮挡的鲁棒性
- 参数极少

### Mixup

混合两张图像和标签。

优势：

- 改进模型的线性性
- 提高泛化能力

注意：

- 在目标检测中，标签混合需要特殊处理
- 通常只混合背景较多的样本

### CutMix

剪切一张图像的随机矩形区域，粘贴到另一张图像上。

优势：

- 比 Mixup 更强的约束
- 保留更多的原始信息

## 自动增强（AutoAugment）

自动搜索最优的增强策略。

优点：

- 无需手工设计
- 通常能找到更好的组合

缺点：

- 计算成本高
- 需要大量数据进行搜索

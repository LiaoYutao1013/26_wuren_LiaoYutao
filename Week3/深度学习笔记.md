## ResNet （CNN的改进变体）

## 1. 背景

传统的CNN中有以下几个问题：

1. 梯度消失：深层网络梯度传播困难，存在梯度变化极其缓慢的情况；
2. 特征退化：网络层数过多反而会降低性能，预测准确度下降；

数学原理：
$$L = f_n(f_{n-1}(...f_1(x)))$$

反向传播时，
$$\frac{\partial L}{\partial x} = \frac{\partial L}{\partial f_n} \cdot \frac{\partial f_n}{\partial f_{n-1}} \cdot ... \cdot \frac{\partial f_1}{\partial x}$$

如果 $\frac{\partial f_i}{\partial f_{i-1}} < 1$，梯度会越乘越小，也就是层数越深，梯度衰减越严重，对模型性能影响很大。

## 2. 残差学习（ResNet的思想）

基本思想是学习残差比直接学习映射更容易。

传统网络中，
$$y = H(x)$$

而在ResNet中，
$$y = H(x) + x$$

其中 $H(x)$ 学习残差，即 $H(x) = y - x$。

数学原理：

设 $y = x + H(x)$，反向传播时，

$$\frac{\partial L}{\partial x} = \frac{\partial L}{\partial y} \left(1 + \frac{\partial H}{\partial x}\right)$$

其中每层梯度都大于1，即使 $\frac{\partial H}{\partial x}$ 很小，+1后也能保证梯度不会衰减（消失）。

## 3. ResNet 架构

### 3.1 残差块（Residual Block）

#### 基础残差块（Basic Block）

(此处让AI作了一个字符画)

```text
┌─ x ─┐
│     │
▼     │
Conv  │ (out_channel = in_channel)
ReLU  │
Conv  │
│     │
└─+─┬─┘ (相加)
  │ │
  ▼ ▼
  H(x) + x
    │
    ▼
  ReLU
```

数学表达式为
$$y = \text{ReLU}(x + F(x))$$
这里ReLU不只处理x，还处理了残差函数$F(x)$，达到学习残差的效果。

#### 瓶颈残差块（Bottleneck Block）

用于更深的网络（50 层+），减少计算量：

```text
┌─ x ──────┐
│          │
▼          │
Conv 1×1   │ (降维，减少通道)
ReLU       │
Conv 3×3   │
ReLU       │
Conv 1×1   │ (升维，恢复通道)
│          │
└─+─┬──────┘ (相加)
  │ │
  ▼ ▼
  y + x
    │
    ▼
  ReLU
```

## 6. 改进变体

**改进**：将 Batch Norm 和 ReLU 放在卷积前：

```text
原始顺序：Conv → BN → ReLU → Conv → BN
改进顺序：BN → ReLU → Conv → BN → ReLU → Conv
```

## MLP Mixer 与混合架构

MLP-Mixer 证明了视觉模型不一定必须使用卷积或注意力，只要有合适的 token 混合和通道混合，也能学习图像表示。

## MLP-Mixer 结构

```text
Image
  ↓
Patch Embedding
  ↓
Mixer Block × N
  ├─ Token-mixing MLP
  └─ Channel-mixing MLP
  ↓
Classifier
```

## 两类混合

### Token Mixing

沿空间 token 维度混合信息，相当于让不同 patch 之间通信。

### Channel Mixing

沿通道维度混合信息，相当于在每个位置内部重组特征。

## 与 CNN / Transformer 的对比

| 架构 | 空间交互方式 | 优点 | 局限 |
|------|--------------|------|------|
| CNN | 卷积核局部滑动 | 高效、局部归纳偏置强 | 长距离关系需要堆层 |
| Transformer | 注意力全局交互 | 全局建模强 | 计算量高、需要大数据 |
| MLP-Mixer | MLP 混合 token | 结构简单 | 缺少局部归纳偏置 |

## 混合架构

现代视觉架构往往不是纯 CNN 或纯 Transformer，而是混合设计：
- ConvNeXt：用现代训练策略重新设计 CNN。
- MetaFormer：强调 token mixer 的统一框架。
- MobileViT：把轻量卷积和 Transformer 结合到移动端。
- YOLO 系列：CNN 主干 + PAN/FPN + 注意力或轻量模块。

## 学习价值

MLP-Mixer 不一定是工程首选，但它帮助理解一个问题：模型性能来自哪些因素？

- 局部性是否必要？
- 全局交互是否必要？
- 深度、宽度、归一化和训练策略分别贡献多少？

# 数据增强

数据增强是提升模型性能的关键技术。它通过人工生成新的训练样本，增加数据的多样性，改进模型的泛化能力。

## 基础增强方法

### 1. 几何变换

#### 旋转（Rotation）

**参数**：

- 旋转角度：±5° 到 ±30°（推荐 ±15°）
- 对旋转不敏感的场景（如竖拍）可增大范围

#### 缩放（Scaling）

**参数**：

- 缩放范围：0.8 到 1.2（推荐）
- 帮助模型学习多尺度目标

#### 裁剪（Cropping）

**参数**：

- 裁剪比例：0.7 到 1.0
- 可能丢失目标，谨慎使用

#### 翻转（Flipping）

**参数**：

- 水平翻转：通常总是使用（概率 0.5）
- 垂直翻转：取决于场景（如行人检测不推荐）

#### 透视变换（Perspective）

**应用**：自动驾驶（不同视角的车辆）

### 2. 颜色增强

#### 亮度调整

**参数**：

- 范围：0.7 到 1.3
- 模拟不同光照条件

#### 对比度调整

参数范围：0.8 到 1.2

#### 色调调整（HSV 空间）

**参数**：

- H 范围：±0.5
- S 范围：×0.7 到 ×1.3
- V 范围：×0.7 到 ×1.3

#### 高斯模糊

**应用**：模拟运动模糊、焦外

## 高级增强方法

### Cutout

移除图像的随机部分，增强模型的鲁棒性。

**参数**：

- 洞的数量：1-3
- 洞的大小：32×32 到 64×64

**优势**：

- 增强对遮挡的鲁棒性
- 参数极少

### Mixup

混合两张图像和标签。

**优势**：

- 改进模型的线性性
- 提高泛化能力

**注意**：

- 在目标检测中，标签混合需要特殊处理
- 通常只混合背景较多的样本

### CutMix

剪切一张图像的随机矩形区域，粘贴到另一张图像上。

**优势**：

- 比 Mixup 更强的约束
- 保留更多的原始信息

## 自动增强（AutoAugment）

自动搜索最优的增强策略。

**优势**：

- 无需手工设计
- 通常能找到更好的组合

**缺点**：

- 计算成本高
- 需要大量数据进行搜索

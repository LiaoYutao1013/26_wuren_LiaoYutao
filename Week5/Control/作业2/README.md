# 作业2 README

这次作业的主线是：机器狗从 `(0, 0)` 出发，在二维平面中寻找位于 `(3, 3)` 的信号源。代价函数为

```text
f(x, y) = 1/2 * (x - 3)^2 + 10/2 * (y - 3)^2
```

其中 `y` 方向的权重更大，所以同样偏离 1 个单位时，`y` 方向会被惩罚得更重。

## 文件说明

- `src/main.cpp`：用 C++ 和 Eigen 完成 Q1 梯度下降实验，给出 Q2 的 KKT 解，并打印 Q3 的 QP 矩阵。
- `scripts/q3_osqp.py`：用 OSQP 实际求解 Q3 的二次规划。
- `CMakeLists.txt`：C++ 构建配置。若系统没有安装 Eigen，会通过 CMake `FetchContent` 拉取 Eigen 3.4.0。
- `requirements.txt`：Python OSQP 验证脚本所需依赖。
- `机器狗寻优问题.pdf`：原始作业说明。

## Q1：无约束梯度下降

令状态量为

```text
X = [x, y]^T
```

梯度为

```text
grad f(X) = [x - 3, 10(y - 3)]^T
```

梯度下降迭代公式：

```text
X_{k+1} = X_k - eta * grad f(X_k)
```

从 `(0, 0)` 出发，停止条件设置为：

```text
||X - [3, 3]^T|| < 1e-3
```

本机运行结果如下：

| eta | 迭代次数 | 最终 x | 最终 y | 误差 | 状态 |
| --- | ---: | ---: | ---: | ---: | --- |
| 0.01 | 797 | 2.999004 | 3.000000 | 9.963e-04 | 收敛 |
| 0.05 | 157 | 2.999046 | 3.000000 | 9.544e-04 | 收敛 |
| 0.09 | 85 | 2.999010 | 3.000000 | 9.900e-04 | 收敛 |
| 0.10 | 76 | 2.999001 | 3.000000 | 9.990e-04 | 收敛 |
| 0.15 | 50 | 2.999113 | 3.000000 | 8.873e-04 | 收敛 |
| 0.18 | 41 | 2.999122 | 3.000319 | 9.341e-04 | 收敛 |
| 0.181818 | 42 | 2.999344 | 2.999344 | 9.275e-04 | 收敛 |
| 0.19 | 76 | 3.000000 | 2.999001 | 9.990e-04 | 收敛 |
| 0.20 | 100000 | 3.000000 | 0.000000 | 3.000e+00 | 不收敛 |

我的观察是：这个目标函数的 Hessian 为 `diag(1, 10)`，最大特征值是 `10`，固定学习率梯度下降的稳定范围大致为

```text
0 < eta < 2 / 10 = 0.2
```

当 `eta` 接近 `0.2` 时，`y` 方向会明显振荡；当 `eta=0.2` 时，`y` 方向的误差比例为 `1 - 10 * eta = -1`，误差只会正负翻转，不会变小。所以它看起来一直在动，但其实没有真正靠近目标点。

## Q2：有约束最优化与 KKT 条件

加入约束：

```text
x + y <= 4
```

无约束最优解是 `(3, 3)`，但它满足 `x + y = 6 > 4`，不可行。因此最优点应该落在边界：

```text
x + y = 4
```

写成标准不等式：

```text
g(x, y) = x + y - 4 <= 0
```

拉格朗日函数：

```text
L(x, y, mu) = 1/2(x - 3)^2 + 5(y - 3)^2 + mu(x + y - 4)
```

KKT 条件为：

```text
x - 3 + mu = 0
10(y - 3) + mu = 0
mu >= 0
x + y <= 4
mu(x + y - 4) = 0
```

由于约束激活，有：

```text
x + y = 4
```

联立：

```text
x = 3 - mu
y = 3 - mu / 10
x + y = 4
```

解得：

```text
mu = 20 / 11
x* = 13 / 11 = 1.181818...
y* = 31 / 11 = 2.818181...
```

这个结果也可以作为 Q3 OSQP 输出的校验基准。

## Q3：转化为 OSQP 二次规划

OSQP 的标准形式为：

```text
min 1/2 * X^T P X + q^T X
s.t. l <= A X <= u
```

把原目标函数展开：

```text
f(x, y)
= 1/2(x - 3)^2 + 5(y - 3)^2
= 1/2 x^2 + 5y^2 - 3x - 30y + constant
```

常数项不影响最优解，所以取：

```text
X = [x, y]^T

P = [1   0
     0  10]

q = [-3
     -30]
```

约束 `x + y <= 4` 写成：

```text
A = [1  1]
l = [-inf]
u = [4]
```

Python OSQP 输出：

```text
Q3 OSQP result
status: solved
x*: [1.1818181818, 2.8181818182]
objective f(x*): 1.8181818182
x + y - 4: -8.589e-12
KKT reference: [1.1818181818, 2.8181818182]
difference: 2.386e-11
```

OSQP 的数值结果和 Q2 的 KKT 手算结果一致，说明 QP 矩阵写法是对的。

## 学习记录

一开始我直觉上会觉得“离 `(3, 3)` 最近的可行点”应该是约束线 `x + y = 4` 上的中点附近，也就是 `(2, 2)`。但这个问题的目标函数不是普通的欧氏距离，`y` 方向的权重是 `10`，所以优化器更愿意保住 `y`，把更多“让步”放到 `x` 上。最后得到的点是 `(1.1818, 2.8182)`，确实离 `y=3` 很近。

Q1 的学习率实验也挺直观：学习率太小会很慢；学习率接近稳定边界时，某些方向会来回振荡。这个例子虽然只有二维，但已经能看到二次函数曲率对优化算法的影响。

这次和 AI 的对话主要做了三件事：先从 PDF 里提取题目要求，再一起推导 Q1/Q2/Q3 的数学形式，最后把推导整理成可以运行的代码和 README。AI 在这里比较像一个随手可问的助教，适合用来检查符号、矩阵维度和结果是否互相对得上。
